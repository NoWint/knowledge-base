# 个人知识库/学习系统 - 架构设计计划

## 项目概述
构建一个功能完善的个人知识库和学习系统，支持初中全学科，包含题库、内容学习、知识图谱、数据分析等核心功能。

## 部署架构设计

### 纯前端静态架构

**核心策略：100% 纯前端，零后端依赖**

所有数据存储在浏览器 IndexedDB，完全无需服务器。

**架构组成**：
```
GitHub Pages（纯静态前端）
    ↓
浏览器本地存储
    ├── IndexedDB（主数据库）
    └── LocalStorage（轻量配置）
```

**优势**：
- ✅ 零后端依赖，完全静态
- ✅ 100% 免费部署
- ✅ 数据完全本地，隐私性好
- ✅ 无需任何云服务

**数据管理策略**：
- 导出/导入 JSON 备份文件
- 支持浏览器间迁移
- 定期提醒用户备份

### 前端配置
- Next.js 静态导出（`output: 'export'`）
- 纯 Client Components（无 SSR）
- 所有数据操作在客户端完成

## 技术架构

### 前端技术栈
- **框架**: Next.js 15 (App Router, Static Export) + React 19 + TypeScript

## 核心功能模块

### 模块1: 本地用户系统（多账号切换）
**功能点**:
- 本地多用户创建（无需注册登录）
- 用户切换（下拉菜单快速切换）
- 每个用户独立数据空间
- 用户头像/昵称设置
- 删除用户（同时清除数据）

**实现方式**:
- 用户信息存储在 IndexedDB
- 每个用户有独立的数据命名空间
- 通过 `currentUser` 标识切换

### 模块2: 学科目录与内容系统
**功能点**:
- 学科整体目录（与教材同步）
- 知识点分类树
- 文章内容（教学/概述/学习方法）
- 思维导图展示
- 真题资料
- 中考政策解读

**数据库表**:
- `subjects` - 学科信息
- `chapters` - 章节/目录
- `knowledge_points` - 知识点
- `articles` - 文章内容
- `mindmaps` - 思维导图数据
- `policies` - 政策文档

### 模块3: 题库与练习系统
**功能点**:
- 题库管理（按学科、章节、知识点、难度分类）
- 多种练习模式：
  - 自由练习
  - 专项训练（特定知识点）
  - 模拟考试（限时）
  - 错题重做
  - 闯关模式
- 答题界面（支持多种题型）
- 即时反馈与解析
- 订正功能（标记订正状态）
- 重做功能

**题型支持**:
- 单选题
- 多选题
- 填空题
- 判断题
- 解答题（支持文本答案）

**数据库表**:
- `questions` - 题目
- `question_options` - 选项
- `question_knowledge` - 题目-知识点关联
- `test_papers` - 试卷
- `test_paper_questions` - 试卷-题目关联
- `user_answers` - 用户答题记录
- `wrong_questions` - 错题本

### 模块4: 学习数据分析
**功能点**:
- 正确率统计（整体/按学科/按知识点）
- 成绩趋势图表
- 错题列表与分类
- 薄弱知识点分析
- 学习时长统计
- 学习成就系统

**数据库表**:
- `statistics` - 统计数据
- `achievements` - 成就记录

### 模块5: 知识关联图谱
**功能点**:
- 知识点关联关系（前驱、后继、相关）
- 跨学科知识关联
- 可视化知识图谱
- 扩展内容推荐
- 关联题目推荐

**数据库表**:
- `knowledge_relations` - 知识关联关系
- `knowledge_extensions` - 扩展内容

### 模块6: 文件资料管理系统
**功能点**:
- 文件上传到 IndexedDB（支持PDF、Word、图片、视频等）
- 文件夹管理（创建、重命名、移动、删除）
- 文件分类标签系统
- 文件预览（PDF、图片、Markdown）
- 与知识点关联（文件可关联到特定知识点）
- 批量操作（批量上传、批量移动）
- 搜索与筛选（按名称、类型、标签、关联知识点）
- 导出/导入单个文件或文件夹

**IndexedDB 存储结构**:
```
文件以 Blob 形式存储在 IndexedDB 'files' store
元数据存储在 'file_metadata' store:
├── id: 唯一标识
├── userId: 所属用户
├── fileName: 文件名
├── fileType: 文件类型
├── fileSize: 文件大小
├── folderId: 所属文件夹
├── tags: 标签数组
├── knowledgePointId: 关联知识点
└── createdAt: 创建时间

文件夹存储在 'file_folders' store:
├── id: 唯一标识
├── userId: 所属用户
├── folderName: 文件夹名
├── parentId: 父文件夹（根目录为null）
└── createdAt: 创建时间
```

**大文件处理**:
- 使用分块存储避免内存问题
- 图片自动压缩（可选）
- 视频等超大文件建议外部链接

### 模块7: 数据导入导出系统
**功能点**:
- 全量数据导出为 JSON 文件
- 从 JSON 文件导入数据
- 增量导入（合并数据）
- 按模块导出（仅导出题库/仅导出错题等）
- 数据迁移向导（浏览器间迁移）

### 模块8: 智能功能（后期扩展）
**功能点**:
- 基于错题的智能推荐
- 艾宾浩斯遗忘曲线复习提醒
- 自适应难度调整
- 学习路径规划

## IndexedDB 数据库设计

### 架构说明
使用 Dexie.js 封装 IndexedDB，提供类型安全的数据库操作。
所有数据存储在浏览器本地，通过 `userId` 隔离不同用户数据。

### 核心关系模型
```
users (本地用户)
  └── userProfiles
       ├── userSubjects
       └── chapters (全局数据)
            └── knowledgePoints
                 ├── articles
                 ├── questions
                 │    └── userAnswers (按 userId 隔离)
                 └── knowledgeRelations (跨学科)
       
userFiles (IndexedDB Blob)
  └── fileMetadata (按 userId 隔离)
       └── fileKnowledgeLinks

### IndexedDB Store 详细结构（使用 Dexie.js）

```typescript
import { Dexie, type EntityTable } from 'dexie';

const db = new Dexie('KnowledgeBaseDB') as Dexie & {
  users: EntityTable<User, 'id'>;
  userProfiles: EntityTable<UserProfile, 'id'>;
  userSubjects: EntityTable<UserSubject, 'id'>;
  subjects: EntityTable<Subject, 'id'>;
  chapters: EntityTable<Chapter, 'id'>;
  knowledgePoints: EntityTable<KnowledgePoint, 'id'>;
  articles: EntityTable<Article, 'id'>;
  questions: EntityTable<Question, 'id'>;
  questionOptions: EntityTable<QuestionOption, 'id'>;
  userAnswers: EntityTable<UserAnswer, 'id'>;
  wrongQuestions: EntityTable<WrongQuestion, 'id'>;
  testPapers: EntityTable<TestPaper, 'id'>;
  testPaperQuestions: EntityTable<TestPaperQuestion, 'id'>;
  knowledgeRelations: EntityTable<KnowledgeRelation, 'id'>;
  userFiles: EntityTable<UserFile, 'id'>;
  fileFolders: EntityTable<FileFolder, 'id'>;
  fileTags: EntityTable<FileTag, 'id'>;
  fileKnowledgeLinks: EntityTable<FileKnowledgeLink, 'id'>;
};

db.version(1).stores({
  users: 'id, name, createdAt',
  userProfiles: 'id, userId, currentGrade',
  userSubjects: 'id, userId, subjectId',
  subjects: 'id, name, gradeLevel, orderIndex',
  chapters: 'id, subjectId, parentId, orderIndex',
  knowledgePoints: 'id, chapterId, name, difficulty',
  articles: 'id, knowledgePointId, type, createdAt',
  questions: 'id, knowledgePointId, type, difficulty',
  questionOptions: 'id, questionId, label, isCorrect',
  userAnswers: 'id, userId, questionId, answeredAt, isCorrect',
  wrongQuestions: 'id, userId, questionId, status, lastWrongAt',
  testPapers: 'id, name, subjectId',
  testPaperQuestions: 'id, testPaperId, questionId, orderIndex',
  knowledgeRelations: 'id, sourceKpId, targetKpId, relationType',
  userFiles: 'id, userId, folderId, fileName, fileType, createdAt',
  fileFolders: 'id, userId, parentId, folderName',
  fileTags: 'id, userId, tagName',
  fileKnowledgeLinks: 'id, fileId, knowledgePointId',
});
```

  id: string;
  name: string;
  icon: string;
  gradeLevel: string;
  description: string;
  orderIndex: number;

interface Chapter {
  id: string;
  subjectId: string;
  parentId: string | null;
  name: string;
  orderIndex: number;

  chapterId: string;
  name: string;
  description: string;

### 数据隔离机制

```typescript
// 所有用户数据查询都通过 userId 过滤
const getUserAnswers = async (userId: string) => {
  return db.userAnswers.where('userId').equals(userId).toArray();
};

// 用户切换时更新全局状态
const switchUser = async (userId: string) => {
  const user = await db.users.get(userId);
  if (user) {
    localStorage.setItem('currentUserId', userId);
    useUserStore.setState({ currentUser: user });
  }
};
```

## 长线项目特性设计

### 可扩展性
- **模块化架构**: 各功能模块独立，可按需添加/删除
- **数据模型扩展**: Dexie.js 版本升级机制，支持后期添加新 store
- **插件化设计**: 未来可接入第三方功能模块
- **版本化数据**: 支持数据结构迁移和升级

### IndexedDB 版本升级

```typescript
// 版本1：基础数据结构
db.version(1).stores({
  users: 'id, name',
  userProfiles: 'id, userId',
  // ...
});

// 版本2：添加文件管理
db.version(2).stores({
  userFiles: 'id, userId, folderId',
  fileFolders: 'id, userId, parentId',
}).upgrade(trans => {
  return trans.table('users').toCollection().modify(user => {
    user.createdAt = new Date();
  });
});
```

### 数据迁移策略
- **Schema 版本控制**: Dexie.js 版本升级机制
- **数据备份**: 定期导出 JSON 备份文件
- **向后兼容**: 新字段使用默认值，不破坏旧数据

### 配置化管理
```typescript
// config/app.ts
export const appConfig = {
  database: {
    name: 'KnowledgeBaseDB',
    version: 1,
  },
  storage: {
    maxFileSize: 50 * 1024 * 1024, // 50MB
    allowedTypes: ['pdf', 'doc', 'docx', 'png', 'jpg', 'mp4'],
  },
  export: {
    format: 'json',
    includeFiles: false, // 是否包含文件数据
  }
}
```

### 离线支持
- **100% 离线可用**: 所有功能完全离线
- **PWA 支持**: 可安装到桌面/手机
- **自动保存**: 所有操作实时保存到 IndexedDB

## 安全与隐私

### 数据安全
- **本地存储**: 数据完全在用户设备中
- **用户隔离**: 多用户通过 `userId` 逻辑隔离
- **浏览器限制**: 各网站 IndexedDB 互相隔离

### 隐私保护
- **零数据收集**: 不向任何服务器发送数据
- **本地加密**: 可选加密敏感数据（使用 Web Crypto API）
- **完全控制**: 用户随时可导出/清除所有数据
- **浏览器安全**: 利用浏览器沙盒机制

## 技术决策

### 为什么选择 Next.js？
- 支持静态导出（`output: 'export'`）兼容 GitHub Pages
- 文件系统路由，易于组织
- 组件化开发
- 生态丰富

### 为什么选择 Dexie.js（IndexedDB）？
- **零依赖**: 无需任何服务器或云服务
- **大容量**: 浏览器可存储数百MB数据
- **类型安全**: TypeScript 完整支持
- **版本管理**: 内置数据库升级机制
- **查询丰富**: 支持索引、过滤、排序

### 为什么选择 shadcn/ui？
- 可定制性强
- 不是传统组件库，而是代码片段
- 完全控制样式
- 现代化设计

### 为什么使用纯前端？
- GitHub Pages 仅支持静态站点
- 零后端成本
- 数据完全本地，隐私性极佳
- 100% 离线可用

## 扩展性考虑

### 多人使用
- **本地多用户**: 通过 IndexedDB 实现本地多账号
- **用户切换**: 快速切换不同用户
- **数据隔离**: 每个用户独立的数据空间
- **未来扩展**: 如需要云端同步，可接入 Supabase

### 多端适配
- **响应式设计**: 支持桌面/平板/手机
- **PWA支持**: 可安装到桌面/手机
- **触控优化**: 移动端手势操作

### 数据导入导出
- **全量导出**: 一键导出所有数据为 JSON
- **按模块导出**: 可选择导出题库/错题/文件等
- **Excel导入**: 支持从 Excel 导入题库
- **Markdown导入**: 支持导入文章内容
- **浏览器迁移**: 通过导出/导入实现跨浏览器迁移
