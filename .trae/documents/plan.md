## 🎯 项目目标

构建一个 **纯前端、零后端** 的个人知识库和学习系统：
- 支持初中全学科（语文、数学、英语、物理、化学、生物、历史、地理、政治）
- 包含题库系统、错题本、数据分析、文件管理、知识图谱
- 部署于 GitHub Pages，100% 免费
- 数据存储在浏览器 IndexedDB，完全本地化

---

## 🏗️ 技术架构

```
┌─────────────────────────────────────────┐
│         GitHub Pages (静态前端)          │
├─────────────────────────────────────────┤
│  Next.js 15 + React 19 + TypeScript     │
│  Tailwind CSS + shadcn/ui               │
│  Framer Motion (动画)                    │
│  Zustand (状态管理)                      │
├─────────────────────────────────────────┤
│  Dexie.js (IndexedDB 封装)               │
│  ├─ 用户数据 (按 userId 隔离)            │
│  ├─ 全局数据 (学科/章节/知识点/题目)      │
│  └─ 文件数据 (Blob 存储)                 │
├─────────────────────────────────────────┤
│  浏览器本地存储 (IndexedDB + LocalStorage)│
└─────────────────────────────────────────┘
```

## 🔑 关键决策记录

### 决策 1: 纯前端架构
- **选择**: 100% 纯前端，使用 IndexedDB
- **原因**: GitHub Pages 仅支持静态站点，零成本
- **影响**: 数据不跨设备，需导出/导入迁移

### 决策 2: 本地多用户
- **选择**: IndexedDB 逻辑隔离，非认证系统
- **原因**: 无后端，无法实现真正认证
- **影响**: 多设备需分别创建用户

### 决策 3: Next.js 静态导出
- **选择**: `output: 'export'` 配置
- **原因**: 兼容 GitHub Pages
- **影响**: 不能使用 SSR/Server Actions

### 决策 4: Dexie.js 数据库
- **选择**: Dexie.js 封装 IndexedDB
- **原因**: 类型安全、版本管理、查询丰富
- **影响**: 需处理浏览器存储限制
